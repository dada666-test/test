# DOS命令

**help  命令：可以展示当前命令的其他参数**

- dir显示该目录下的所有文件

  - dir /p 分页展示内容
  - dir  /d 展示文件名

- cd切换路径

  - D: 切换到D盘
  - cd  目录名
  - cd  ..返回上一级
  - cd \返回根目录

- tab可以实现文件名的自动补全

- md 创建文件夹

- rd  删除目录

- del  haha.txt   删除文件

- copy  复制

  - copy  被复制文件   目标路径  ----（复制）
  - copy  被复制文件   新文件名字  ----（复制且重命名）

- ren  修改文件名

- ```
  >文件名  将命令结果保存到文件中
  ```

- cls 清屏

- .bat  批处理文件

- time 时间

- date  日期

- |  命令管道符

  - 命令1|命令2|命令3    **后一个命令是对前一个命令的结果进行处理**

- tree  展示目录的结构

  - tree  /f详细展示目录结构



# 网络体系

计算机网络是用**通信设备和线路**将分散在不同地点的有独立功能的计算机系统相互连接起来，并**按照网络协议**进行数据通信，实现资源共享的计算机集合

按地域范围分

- 局域网LAN
- 广域网WAN
- 城域网MAN

网络协议：网络协议为计算机网络中进行**数据交换而建立的规则、标准或约定的集合。**

网络分层

- OSI七层：物理层，数据链路层，网络层，传输层，会话层，表示层，应用层
- TCP/IP五层：物理层，数据链路层，网络层，传输层，应用层
- TCP/IP四层：网络接口层，网络层，传输层，应用层

- 优点：
  - 使复杂系统简化
  - 易于维护、系统的更新
  - 某层功能变化，不会影响系统其余部分
- 缺点：
  - 有些功能可能在不同层重复出现
  - 某层的功能可能需要仅存在其他某层的信息

## 物理层

- 物理层是OSI模型中最底层，负责建立、维护和断开物理连接。
- 它传输的数据单位是比特（Bit）。
- 典型的物理层设备有光纤、同轴电缆、双绞线、中继器和集线器。

## 数据链路层

- 数据链路层在物理层提供的比特流服务基础上，将比特**组装成帧（Frame）**，并负责帧的传输。
- 功能包括物理地址寻址、数据的成帧、流量控制、数据的检错和重发。
- **数据链路层确保信息在物理链路上可靠地传输**。
- 典型设备包括二层交换机、网桥和网卡。
-  HDLC, PPP, IEEE 802.3

## 网络层

- 网络层负责为数据包选择路由和交换节点，确保数据及时传送。
- 网络层的主要功能是通过逻辑地址（如IP地址）进行路由选择，确保数据能够跨越多个网络（不仅仅是两个直接相连的网络）到达目的地。
- 典型设备是路由器。
- IP

## 传输层

- 传输层提供**端到端**的数据传输服务，确保数据的**完整性和可靠性**。
- 它负责将上层应用发来的数据进行分割，并将它们传输到网络层。
- 在接收端，传输层负责将分割的数据重新组装成完整的数据。
- 传输层使用端口号来标识不同的应用程序。
- TCP，UDP

## 会话层

- 会话层负责在网络中的两节点之间**建立、维护和终止通信**。
- 它提供的服务包括建立和维护会话，以及数据交换的同步。

## 表示层

- 表示层负责数据的表示、安全和压缩。它将数据转换为能够与接收方的系统格式兼容并适合传输的格式。
- 表示层的功能包括数**据格式转换、数据加密和解密，以及数据压缩。**

## 应用层

- 应用层是OSI模型的最高层，直接为**用户的应用程序提供服务。**
- 它负责处理特定的应用程序细节，如文件传输、电子邮件和其他网络软件服务。应用层使用的数据单位是报文。
- 协议：FTP、TFTP、NFS、SMTP、POP3、HTTP、DNS
